# 工具评估器

这个Skill用于评估Agent可用工具/技能的边界和可用性。

## 何时调用

在以下情况下调用此Skill：
- 用户要求"评估工具"
- 用户要求"技能摸底"
- 用户要求"测试工具可用性"
- 用户要求"评估工具边界"
- 用户提到工具相关测试

## 核心功能

### 1. 工具边界评估

**评估流程：**

1. **接收Case批次**
   - 从用户提供case列表
   - 或自动生成测试case
   - 按优先级排序

2. **逐个执行测试**
   - 调用工具/技能执行
   - 记录执行结果
   - 分析返回质量

3. **边界探测**
   - 测试参数限制
   - 测试输入范围
   - 测试输出格式
   - 测试错误处理

4. **生成评估报告**
   - 结构化输出
   - 包含优化建议
   - 记录问题点

### 2. 评估维度

**评估工具的多个维度：**

#### 功能维度
- **核心功能**：工具的主要功能是否正常
- **边界条件**：极端情况下的表现
- **错误处理**：异常情况的处理能力

#### 性能维度
- **响应时间**：执行速度
- **资源消耗**：内存、CPU使用
- **并发能力**：多任务处理能力

#### 质量维度
- **返回质量**：输出结果的准确性
- **稳定性**：多次执行的一致性
- **可靠性**：成功率

#### 易用性维度
- **参数设计**：参数是否合理
- **文档完整性**：文档是否清晰
- **错误提示**：错误信息是否友好

## 评估流程

### 步骤1：准备阶段

```
1. 收集工具信息
   - 工具名称和描述
   - 参数定义
   - 预期功能

2. 设计测试Case
   - 正常case
   - 边界case
   - 异常case

3. 准备测试环境
   - 检查依赖
   - 配置参数
```

### 步骤2：执行阶段

```
1. 执行测试Case
   - 记录输入
   - 记录输出
   - 记录时间

2. 收集数据
   - 成功/失败次数
   - 错误信息
   - 性能数据

3. 分析结果
   - 对比预期
   - 识别问题
   - 分析原因
```

### 步骤3：报告阶段

```
1. 生成报告
   - 汇总数据
   - 分析问题
   - 提出建议

2. 输出结果
   - 结构化报告
   - 问题列表
   - 优化建议
```

## 测试Case设计

### 正常Case
- 标准输入
- 预期输出
- 验证功能

### 边界Case
- 极端值
- 边界条件
- 压力测试

### 异常Case
- 错误输入
- 缺失参数
- 格式错误

## 评估指标

### 定量指标
- **成功率**：成功次数 / 总次数
- **平均响应时间**：总时间 / 次数
- **错误率**：错误次数 / 总次数

### 定性指标
- **易用性**：主观评分
- **文档质量**：主观评分
- **错误提示**：主观评分

## 最佳实践

### 1. 全面覆盖
- 测试所有功能点
- 覆盖所有参数组合
- 测试各种场景

### 2. 数据驱动
- 基于实际数据测试
- 记录详细日志
- 分析统计数据

### 3. 问题追踪
- 记录所有问题
- 分类问题类型
- 追踪修复状态

### 4. 持续评估
- 定期重新评估
- 对比历史数据
- 追踪改进效果

## 输出内容

此Skill将生成：
- **评估报告**：结构化的评估结果
- **问题列表**：发现的问题清单
- **优化建议**：改进建议
- **测试数据**：原始测试数据

## 注意事项

- **客观公正**：基于事实评估
- **详细记录**：记录所有细节
- **可重现**：测试可重现
- **持续更新**：定期更新评估
