# Agent进化计划心跳提醒

## 提醒目的

本文件用于定时提醒AI系统关于Agent自我进化项目的进展情况，确保进化计划能够持续推进，避免遗忘关键任务和里程碑。

## 提醒频率

- **每日提醒**：检查当日任务进展，更新工作日志
- **每周提醒**：回顾周进展，调整下周计划
- **每月提醒**：评估月度目标完成情况，规划下月工作
- **每季度提醒**：进行季度评估，调整长期计划

## 核心提醒内容

### 1. 进化计划状态

- [ ] 当前迭代进度：检查当前迭代的任务完成情况
- [ ] 进化指标：评估Agent性能提升情况
- [ ] 经验池更新：确认经验池是否及时更新
- [ ] 测试反馈：检查测试结果和反馈数据

### 2. 关键任务提醒

- [ ] 核心引擎优化：确保生成器、执行器和进化器的性能
- [ ] Agent管理：检查Agent存储、版本控制和元数据管理
- [ ] 任务管理：确认任务定义、调度和监控系统的运行
- [ ] 评估系统：验证评估指标和反馈机制的有效性
- [ ] 工具库：确保工具库的完整性和可用性

### 3. 进化策略检查

- [ ] 基于反馈的进化：确认测试反馈数据是否用于指导进化
- [ ] 多代竞争：检查是否生成多个Agent变体进行竞争
- [ ] 知识迁移：验证成功Agent的知识是否迁移到新Agent
- [ ] 渐进式优化：确认是否进行逐步、增量式的改进

### 4. 自我改进机制

- [ ] 自评估：检查Agent是否定期评估自身性能
- [ ] 自诊断：确认是否识别自身的弱点和改进机会
- [ ] 自优化：验证是否自动生成和测试改进方案
- [ ] 自学习：检查是否从执行经验中学习并调整行为

### 5. 经验池管理

- [ ] 经验收集：确认是否收集和整理经验数据
- [ ] 经验分类：检查经验是否按技术领域、问题类型等分类
- [ ] 经验应用：验证经验是否用于LLM训练和Agent进化
- [ ] 经验更新：确认经验池是否定期更新

### 6. 测试管理

- [ ] 测试case设计：检查是否设计了全面的测试case
- [ ] 测试执行：确认测试是否按计划执行
- [ ] 反馈收集：验证是否收集和分析测试反馈
- [ ] 优化方案：检查是否基于测试结果生成优化方案

## 触发时间设置

### 每日提醒
- **时间**：每天上午9:00
- **内容**：检查当日任务，更新工作日志

### 每周提醒
- **时间**：每周一上午9:00
- **内容**：回顾上周进展，规划本周任务

### 每月提醒
- **时间**：每月1日上午9:00
- **内容**：评估上月目标完成情况，规划下月工作

### 每季度提醒
- **时间**：每季度首月1日上午9:00
- **内容**：进行季度评估，调整长期计划

## 操作指南

1. **触发提醒**：按照设定的时间触发心跳提醒
2. **检查状态**：根据提醒内容检查各模块的状态
3. **更新记录**：记录检查结果和发现的问题
4. **调整计划**：根据检查结果调整进化计划
5. **生成报告**：定期生成进化进展报告

## 紧急提醒

当出现以下情况时，应立即触发紧急提醒：

- Agent性能显著下降
- 系统出现严重错误或故障
- 发现安全漏洞
- 关键功能无法正常运行
